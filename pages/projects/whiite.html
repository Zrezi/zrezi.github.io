whiite linux


Whiite Linux is an old version of a specific Linux distribution that was ported to run on the PowerPC architecture, which is the architecture of the CPU that is in the Wii. Granted, the Wii is very minimal in terms of computer performace, but I believe the distribution was created as a proof of concept — that Linux could be run on the Wii. It took about a week of diving deep into old forum posts when the (albeit very good) tutorial to install Whiite failed. There were many random errors that had little to no documentation. This page is a guide for installing this version of Linux on your Wii with hopefully as little trouble as possible.


before we begin

Keep in mind that the Wii is a pathetic computer in the realm of things. Whiite does not even come with a graphical interface — everything is in command line. If the terminal scares you, run far away. Furthermore, do not expect to run anything spectacular on the machine. If you want to install Whiite for the novelty of running Linux on a Wii, that's perfect. If you plan is to write a high performance webserver in C and use the Wii as your personal hosting machine...think again.


disclaimer

I think this is pretty self-explanatory. You are deliberately messing with the Wii's operating system. If something "goes wrong" in the installation process, you might brick your Wii and render it useless. This is your warning. If you choose to continue I am not responsible for anything that happens to your Wii or your computer.


requirements

In order to install Whiite you're going to need a few things.

  • At least 2 hours
  • 2GB SD card
  • 512MB or larger USB flash drive
  • USB keyboard
  • Computer with -nix installed. A Virtual Machine should work fine if you're on Windows
  • Internet-connected Wii
  • Wiimote
  • TV
  • You're not scared of the terminal
  • Ability to follow directions

technical specs

Like I've said already, the Wii has pretty poor specs. Don't expect to run anything intensive.

  • Broadway 750CL 729.00MHz Processor
  • 76204kB of RAM
  • Mitsumi DWM-W004 WiFi 802.11b/g
    My average tested speeds on a 20Mbit connection were ~160kB/s
  • 18W active power consumption

getting whiite

If you're here that means you actually want to install Whiite on your Wii. That's cool. The next steps will walk you through how to get your Wii ready for Whiite, and take you step by step through the installation process. The entire thing probably takes ~1 hour if nothing goes wrong. Could be quicker too if you can read fast! That reminds me, don't skip steps. If a step is done out of order it will probably cause the installation to fail, and waste your time. Alright, onwards.


updating your wii to 4.3

If you know how to update your Wii, or are already on 4.3, you can skip this step.

  • Start up your Wii, and navigate to the Settings
  • Choose the right option, "Wii System Settings"
  • Scroll right through the options until you find Internet
  • Make sure that your Wii is connected to the internet via a wireless connection by either selecting a previous connection option, or by setting up a new connection point
  • Once you have working WiFi, return to the system settings and choose "Wii System Update" (it's on the third page of options)
  • Wait for the Wii to update. This may take upwards of 10 minutes depending on your internet connection speed
  • After it updates, it should reboot by itself (I think), and return to the main Wii Menu
  • Confirm the update was successful by going back to "Wii System Settings", and in the top right of the menu you should see "Wii System 4.3X", where X is your Wii System country's version

installing homebrew

Before you install Homebrew, you need to make sure your Wii is upgraded to 4.3. Installing homebrew is a somewhat lengthy process itself, and there are countless YouTube tutorials and guides out there. I'd go ahead and follow this, but DO NOT INSTALL BOOTMII YET.


installing ios236

This step reverts the Wii's native operating system to an earlier version, the one that allows us to take advantage of a system exploit and write to the Wii's NAND memory.

  • Download this broken link, and place it in the apps folder on your SD card
  • Navigate back to the Homebrew Channel, and run the installer. Choose the first option when it loads, and wait for it to complete
  • As soon as it's done, you can exit the app and restart the Wii
  • Once restarted, move onto the next step

installing bootmii as boot2

Now that IOS236 has been successfully installed, we'll take advantage of that exploit I mentioned.

  • Go back to the Hackmii bundled installer in your Wii's message board (Letterbomb)
  • Run it, and wait for it to load again
  • Press (1) on the Wiimote, or (A) if you're using a gamecube controller to continue pass the loading screen
  • Choose these options as they appear
    • Install Bootmii
    • Install as boot2
  • Confirm the installation, and you should see some information about NAND memory being written to. This means that it worked
  • If you are unable to install Bootmii at all, that probably means that you didn't do the previous step which was to install IOS236. Make sure you've done that step and then try again
  • After the installation completes, you can exit the bundled installer and move on to setting up the SD card and the USB drive

setting up the sd card and usb

This is the only part that isn't done....I'll get around to finishing it. The Whiite tutorial actually does a decent job at explaining this section of the install.


run the whiite installer

We will be using the installer for the sole purpose of how it partitions your SD card. I have been unsuccessful in trying to manually partition the SD card, even when it matches the layout of how the installer does it.

  • Boot the Wii with the SD, USB drive, and USB keyboard plugged in. Navigate to the Homebrew Channel, and run the gc-whiite installer app
  • A couple menus will appear. Using the keyboard, you're always going to want to say <yes> when presented with that option
  • Eventually, it will start to install. Using the recommended 2GB SD card, this process will take ~15 minutes. Do NOT unplug anything or turn off your Wii during this time

    When the menu appears about "creating the journal", if it appears to freeze that is normal. For some reason this step takes a bit longer than all of the previous steps while installing

    I tried using a 16GB card, and shut down my Wii after an hour of waiting. Just food for thought.
  • When the process completes, it will prompt you to press enter and restart the system. Do this. It won't restart, or it might. Once it turns completely off or you turn it off, move on to the next step

wipe the sd card

Remember I said that we're only using the installer for the way that it partitions the SD card, nothing else. So if it seems weird that you're deleting the thing you just technically installed, the point of the installation was to partition is in a correct way.

  • Unplug the SD card from your Wii, and reconnect it to your computer
  • When plugged in, you should find that two separate partitions have been created by the installer
    • boot
    • whiite
  • Using terminal, the path to the mounted partitions should be something like "/media/$USER/boot/" and "/media/$USER/whiite/"
  • We need to remove all files on both partitions. Using the terminal, run:
    • rm -rf /media/$USER/boot/*
    • rm -rf /media/$USER/whiite/*
  • Confirm that the files have been removed by either ls'ing in terminal, or navigating to the partitions with your file manager

    Note: When unplugging the SD card from your computer, make sure to eject it safely. For me, when performing copy and deletes from the SD from my computer, the terminal commands would end but the SD card would still be performing actions. If you do not eject safely, you risk corrupting the filesystem or the kernel

add bootmii

Now you can copy over the bootmii app boot files. We will be replacing ppcboot.elf however, so we will delete that.

  • Remember the folder you copied from your SD card when you initially installed bootmii? Copy that entire folder into the SD card's boot partition
  • We will be overwriting the ppcboot.elf file in the bootmii folder, so we can delete it first via terminal:
    • rm /media/$USER/boot/bootmii/ppcboot.elf

getting the correct kernel

We want the latest, most updated version of the Whiite kernel. Since bootmii is now reinstalled on the partitioned SD card, we can grab the kernel and copy that over as well.

  • The filestructure of the SD card should look like this:
    • boot (partition)
      • bootmii
        • armboot.bin
        • bootmii.ini
    • whiite (partition)
  • Go to this website and download the kernel version that best fits your TV and Wii setup. I would say 99 out of 100 times you're going to want to use the 5th option, "mikep5-zImage-2.6.32.mini.480p(NTSC).elf"
  • Move the .elf file into the bootmii folder on your SD card, either with the mv terminal command or by dragging and dropping the file
  • Rename the file "ppcboot.elf"
    • mv /media/$USER/boot/bootmii/mikep5-zImage-2.6.32.mini.480p(NTSC).elf /media/$USER/boot/bootmii/ppcboot.elf
    ... taking the place of the bootmii ppcboot.elf generated file that you deleted earlier!

updated filesystem

Now that we have the kernel and the bootloader (which is bootmii) installed, we can install the filesystem. We want to get the latest version, but it's still been 8 years since a new version has come out.

  • Download this file
  • Navigate to the whiite directory on your SD card
    • cd /media/$USER/whiite
  • Untar the downloaded file into this location using the terminal
    • sudo tar -xjvf /home/$USER/Downloads/debian-lenny-5.0+whiite-1.10.tar.bz2
  • Wait for the untarring process to finish, it may take ~10 seconds to complete
  • Eject the drive safely, and wait for confirmation that you're able to unplug before you do so. Again, not safely removing the SD card from your computer risks corruption of the filesystem

once whiite is installed

Alright, so you should'nt've (that's a legit word) turned the Wii on since all of the filesystem and kernel copying. Everything should be copied over to the SD card, no errors should've come up, etc. etc.


booting for the first time

This is really the easiest step. Just turn it on and login.

  • Make sure that the SD card is in the Wii
  • Turn on the Wii
  • You should see a black background with white text scrolling fairly fast. Wait for it to stop.
  • It shouldn't crash. If so, your best bet is to Google the generic error(s), not specifying the Wii
  • When prompted (a.k.a. when the text stops scrolling), enter whiite's default login information:
    • Login: root
    • Password: whiite
  • You should now be logged into whiite on your Wii as root

fixing the keyboard

For some reason the default keyboard layout is in Spanish. Weird.

  • If you try typing some symbols, such as '-', you can see that they don't display the correct characters
  • Run the command dpkg-reconfigure console-data. To type the dash, use the numberpad '-' character. That one works.
  • Wait for the configuration tool to load...
  • Choose these options as they appear to load a standard QWERTY English keyboard
    • OK
    • Select keymap from full list
    • Scroll down with the arrow keys until you find "pc / qwerty / US american / Standard / Standard"
    • Press enter for that choice
  • Wait for the tool to finish working...
  • Now your keyboard should be set up properly

setting up wifi

WiFi is the most finicky part of the entire installation. If this part doesn't work for you, I most likely have no idea why. Either get Googling or reinstall (but reinstalling probably won't fix it...I tried).

  • Whiite comes bundled with an easy installer that automatically modifies the "/etc/network/interfaces" file, aptly called Whiite-ez-wifi-config
  • Navigate to root, cd ~
  • Run ./whiite-ez-wifi-config
  • Follow the onscreen prompts to enter your network SSID and passkey
  • There will be an option to test your connection. TEST THE CONNECTION. If it doesn't work, don't worry. It might've worked anyway and the test is wrong.
  • Exit the installer. Back in the terminal, you can run iwconfig to see if you're connected to a network
  • You'll be looking for the wlan0 connection. If you can't connect to WiFi still, you can manually check the file by running vi /etc/network/interfaces. You can also restart at this time, that might help: shutdown -h now
  • If everything looks like it's working, ping a website like Google and hopefully you'll see some action

getting correct aptitude repositories

Since the version of Linux that Whiite is based on (Debian Lenny) is from nearly a decade ago, the respository paths for Aptitude are obviously broken. Luckily there's an easy fix.

  • Debian Lenny is now unsupported, so any apt-get install will fail due to the wrong repository source being used
  • To update this, use the following commands in order:
    • deb http://archive.debian.org/debian/ lenny contrib main non-free
    • deb http://archive.debian.org/debian-security lenny/updates main
    • apt-get install debian-archive-keyring

upgrade and update aptitude

If you got this far, I probably don't need to show you how to upgrade/update apt.

  • Overall I think it was ~150MB I have no idea how much it was to upgrade and update everything
    • apt-get update
    • apt-get upgrade
    • apt-get update

trying apt

If the previous step didn't fail, then you're all set up. You can't do much with the Wii, but you can look at htop. Speaking of, you can download that.

  • htop is essentially a better version of top, which is your basic System Monitor
  • apt-get install htop
  • Likewise, Debian Lenny comes with the Vi text editor. However, if you're planning to do any text editing at all just get Vim.
  • apt-get install vim

finished

If you're here, you either skipped to the bottom to see what the last few steps were or if you have legitimately gotten to this point, you should have a Wii with 100% working Linux. It's fast, it's pretty, and it's pretty cool.


help

Honestly, there's so many little things that could go wrong during this process, and usually even a minor one will render the kernel or the filesystem unusable, especially not letting your computer copy all of the files over correct. I've installed Whiite on 2 Wiis, and they each had their own individual problems and complications.

If you run into an issue that you can't figure out, I'd recommend just completely wiping the SD card and the USB drive, and starting from scratch. If it locks up at the same step twice, try it a third time and if it still doesn't work then contact me here.
